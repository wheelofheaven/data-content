Wheel of Heaven 以 [CC0](https://creativecommons.org/publicdomain/zero/1.0/) 開源，歡迎貢獻者。項目以一位作者的第一人稱單數聲音進行，但圍繞它的工作——翻譯、編輯、基礎設施、公共討論——受益於眾多雙手。

貢獻者無需贊同項目的框架。批評性的參與是受歡迎的。

## 在開始之前

請閱讀[行為準則](/zh-Hant/code-of-conduct/)。參與項目的公共空間——Discussions、Issue、Pull Request——以此為前提。

請瀏覽 [FAQ](/zh-Hant/faq/)，特別是關於工作假設、源頭家族、以及項目如何處理論斷類型的條目。它們提供的語境會讓大多數貢獻變得容易。

## 貢獻的方式

### 討論與提問

[GitHub Discussions](https://github.com/orgs/wheelofheaven/discussions) 是項目主要的公共論壇。誠實的問題、實質性的異議、替代性的解讀、新的資料來源，都在那裡受歡迎。討論線索有時會直接回饋到語料庫中——一個尖銳的問題，可能最終反映在某個 Wiki 條目或 FAQ 答覆中。

### 翻譯

本站以英文為源頭，發布九種語言。譯文由人與工具共同從英文原文派生，在可能時進行編輯審校。歡迎更多語言；已有譯文也總是可以改進。

如果你對翻譯感興趣，請打開一個 Discussion 話題或 Issue，描述語言與範圍。項目會提供源材料、項目專用術語表，以及風格指南。

### 內容更正與建議

如果站內某頁存在事實性錯誤、失效鏈接、錯誤引用某來源，或對某一學界立場表述失當，項目寧願知情。請在[倉庫](https://github.com/wheelofheaven)開啟 Issue，註明頁面 URL 與具體更正。

解讀上的分歧——你與語料庫對同一來源的不同解讀——應放入 Discussions，而非 Issue。項目歡迎此類分歧，但它與事實性更正屬於不同種類的貢獻。

### 代碼與基礎設施

本站是用 Zola 構建的靜態站點，使用 Tera 模板主題（[Bifrost](https://github.com/wheelofheaven/bifrost)）與 SCSS 樣式。構建在一條公開的 CI 流水線中運行。Pull Request 受歡迎，用於 Bug 修復、可訪問性改進、性能工作，以及契合項目學術語域的小型功能。

對於更大的改動——視覺重設計、導航重構、新的內容版塊——請先開啟一個 Discussion。

### 財務支持

項目目前完全由作者自掏腰包資助。通過 [Open Collective](https://opencollective.com/wheelofheaven) 提供的自願捐助將用於託管、基礎設施與翻譯工作。CC0 許可證意味著任何人——包括項目自身——都無法獨佔地將內容貨幣化。財務支持是真正自願的，只用於讓項目持續運轉。

## 實際中貢獻長什麼樣

一份好的貢獻會說明它在改什麼、為什麼改、依據為何。一個修復某 Wiki 條目失效鏈接的 Pull Request 只需一行描述。一個修訂實質性解讀的 Pull Request 需要更多——至少要給出新解讀的來源，以及一段說明它為何優於現有解讀的文字。

項目的編輯標準同樣適用於貢獻：學術語域、對來源敏感、對論斷類型給出誠實的標註。倉庫旁維護的[內容規則](https://github.com/wheelofheaven)詳細描述了這些標準。

如果你不確定某個貢獻是否合適，請先開啟一個 Discussion。作者會直接告訴你，並可能為該貢獻建議更合適的形式。提問不會受罰。

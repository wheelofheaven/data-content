Wheel of Heaven 以 [CC0](https://creativecommons.org/publicdomain/zero/1.0/) 开源，欢迎贡献者。项目以一位作者的第一人称单数声音进行，但围绕它的工作——翻译、编辑、基础设施、公共讨论——受益于众多双手。

贡献者无需赞同项目的框架。批评性的参与是受欢迎的。

## 在开始之前

请阅读[行为准则](/zh/code-of-conduct/)。参与项目的公共空间——Discussions、Issue、Pull Request——以此为前提。

请浏览 [FAQ](/zh/faq/)，特别是关于工作假设、源头家族、以及项目如何处理论断类型的条目。它们提供的语境会让大多数贡献变得容易。

## 贡献的方式

### 讨论与提问

[GitHub Discussions](https://github.com/orgs/wheelofheaven/discussions) 是项目主要的公共论坛。诚实的问题、实质性的异议、替代性的解读、新的资料来源，都在那里受欢迎。讨论线索有时会直接回馈到语料库中——一个尖锐的问题，可能最终反映在某个 Wiki 条目或 FAQ 答复中。

### 翻译

本站以英文为源头，发布九种语言。译文由人与工具共同从英文原文派生，在可能时进行编辑审校。欢迎更多语言；已有译文也总是可以改进。

如果你对翻译感兴趣，请打开一个 Discussion 话题或 Issue，描述语言与范围。项目会提供源材料、项目专用术语表，以及风格指南。

### 内容更正与建议

如果站内某页存在事实性错误、失效链接、错误引用某来源，或对某一学界立场表述失当，项目宁愿知情。请在[仓库](https://github.com/wheelofheaven)开启 Issue，注明页面 URL 与具体更正。

解读上的分歧——你与语料库对同一来源的不同解读——应放入 Discussions，而非 Issue。项目欢迎此类分歧，但它与事实性更正属于不同种类的贡献。

### 代码与基础设施

本站是用 Zola 构建的静态站点，使用 Tera 模板主题（[Bifrost](https://github.com/wheelofheaven/bifrost)）与 SCSS 样式。构建在一条公开的 CI 流水线中运行。Pull Request 受欢迎，用于 Bug 修复、可访问性改进、性能工作，以及契合项目学术语域的小型功能。

对于更大的改动——视觉重设计、导航重构、新的内容版块——请先开启一个 Discussion。

### 财务支持

项目目前完全由作者自掏腰包资助。通过 [Open Collective](https://opencollective.com/wheelofheaven) 提供的自愿捐助将用于托管、基础设施与翻译工作。CC0 许可证意味着任何人——包括项目自身——都无法独占地将内容货币化。财务支持是真正自愿的，只用于让项目持续运转。

## 实际中贡献长什么样

一份好的贡献会说明它在改什么、为什么改、依据为何。一个修复某 Wiki 条目失效链接的 Pull Request 只需一行描述。一个修订实质性解读的 Pull Request 需要更多——至少要给出新解读的来源，以及一段说明它为何优于现有解读的文字。

项目的编辑标准同样适用于贡献：学术语域、对来源敏感、对论断类型给出诚实的标注。仓库旁维护的[内容规则](https://github.com/wheelofheaven)详细描述了这些标准。

如果你不确定某个贡献是否合适，请先开启一个 Discussion。作者会直接告诉你，并可能为该贡献建议更合适的形式。提问不会受罚。
